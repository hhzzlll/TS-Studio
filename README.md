# TS-Studio

TS-Studio 是一个面向时间序列数据分析、建模实验与预测结果可视化的 Web 应用系统。项目采用前后端分离架构，将数据管理、统计分析、模型训练、预测推理与结果展示整合到统一的交互式平台中，旨在为时间序列预测相关的课程设计、毕业论文实验、科研原型验证和数据分析工作提供可复用的工程支撑。

本项目侧重于系统化实验流程的组织与可视化表达，不在 README 中展开具体算法实现。算法相关说明或源代码可进一步查阅[https://github.com/hhzzlll/TimeDTR](https://github.com/hhzzlll/TimeDTR)。

## 项目定位

TS-Studio 面向具有连续观测特征的数据场景，例如金融汇率、电力负荷、传感器监测、气象观测以及其他多变量时间序列任务。系统通过 Web 化界面降低实验配置、数据处理和结果观察的操作成本，使用户能够在同一平台内完成从数据上传到预测结果分析的完整流程。

项目具有以下特点：

- **工程完整性**：覆盖用户认证、数据集管理、统计分析、训练任务管理、预测任务执行和结果可视化等核心环节。
- **实验可复现性**：训练配置、任务状态、模型记录和预测输出由后端统一管理，便于实验过程追踪。
- **交互可视化**：前端提供图表化展示能力，支持对数据分布、列特征、预测曲线和误差指标进行直观分析。
- **扩展友好性**：后端以 Django REST Framework 组织 API，前端以 Vue 3 组件化开发，便于后续扩展新的数据处理流程、模型接口或可视化模块。

## 功能模块

### 1. 用户与权限管理

系统提供基础用户账户能力，包括注册、登录、登出、账号删除和密码重置。前端通过 Token 方式维护登录态，后端基于 Django REST Framework TokenAuthentication 提供接口认证支持。

### 2. 数据集管理

数据管理模块支持用户上传、预览、下载和删除时间序列数据文件。系统按照用户维度组织数据目录，便于不同用户之间的数据隔离与实验管理。

主要能力包括：

- CSV、XLSX 等表格数据文件管理
- 数据集列表展示
- 数据文件预览
- 数据列信息读取
- 数据文件下载与删除

### 3. 统计分析

统计分析模块面向建模前的数据理解过程，提供描述性统计、列级分析和基础可视化能力。该模块有助于用户观察时间序列数据的尺度、分布、缺失情况、波动特征和变量关系，为后续实验配置提供依据。

### 4. 模型训练管理

训练模块提供模型训练任务的创建、配置、运行状态查询与任务控制能力。训练记录通过数据库保存，包括任务名称、创建时间、运行状态、配置参数、评价指标、日志信息和检查点路径等。

系统支持训练任务的状态跟踪，便于用户管理多个实验版本并进行结果对照。

### 5. 预测与结果展示

预测模块用于基于已完成模型或内置预测方法生成预测结果，并在前端以图表形式展示真实值、预测值及相关评估信息。该模块强调预测结果的可解释呈现，便于用户开展实验报告撰写、误差分析和方法对比。

### 6. 对比实验支持

系统保留传统时间序列预测方法接口，能够服务于对比实验设计。该能力主要用于辅助实验分析和论文结果呈现，使用户能够在统一数据与统一展示环境下观察不同预测方案的表现差异。

## 技术架构

项目采用前后端分离的 Web 系统架构：

```text
TS-Studio
├── frontend/   Vue 3 + Vite 前端应用
├── backend/    Django + Django REST Framework 后端服务
└── .gitignore   
└── README.md
```

### 前端技术栈

| 技术                     | 用途                           |
| ------------------------ | ------------------------------ |
| Vue 3                    | 构建单页应用与交互界面         |
| TypeScript               | 提升前端代码类型约束与可维护性 |
| Vite                     | 前端开发服务器与构建工具       |
| Vue Router               | 页面路由与访问控制             |
| Axios                    | REST API 请求封装              |
| ECharts                  | 时间序列图表与统计可视化       |
| Element Plus / Radix Vue | UI 组件支持                    |
| Tailwind CSS             | 页面样式与布局辅助             |

### 后端技术栈

| 技术                       | 用途                       |
| -------------------------- | -------------------------- |
| Django                     | Web 后端基础框架           |
| Django REST Framework      | RESTful API 构建           |
| SQLite                     | 开发阶段轻量级数据存储     |
| pandas / NumPy             | 数据读取、处理与统计分析   |
| scikit-learn / statsmodels | 实验分析与传统预测方法支持 |
| Matplotlib                 | 后端辅助绘图与分析支持     |

## 环境要求

建议开发环境如下：

- Python 3.10 或更高版本
- Node.js 18 或更高版本
- npm 9 或更高版本
- Windows、Linux 或 macOS 开发环境

如需运行深度学习训练或推理任务，建议根据实际算法依赖额外配置 PyTorch、CUDA 或相关 GPU 环境。

## 后端启动

进入后端目录并安装 Python 依赖：

```bash
cd backend
pip install -r requirements.txt
```

如运行算法相关任务需要额外依赖，可根据实际环境安装：

```bash
pip install einops tensorboardX wandb
```

初始化数据库并启动后端服务：

```bash
python manage.py migrate
python manage.py runserver
```

后端默认服务地址：

```text
http://127.0.0.1:8000
```

## 前端启动

进入前端目录并安装依赖：

```bash
cd frontend
npm install
```

启动开发服务器：

```bash
npm run dev
```

前端默认访问地址通常为：

```text
http://localhost:5173
```

前端开发服务器已在 `vite.config.ts` 中配置 `/api` 代理，请确保后端服务同时运行。

## 数据与实验管理说明

系统默认将用户上传数据保存在后端 `data/` 目录下，并结合用户身份进行组织。训练任务相关记录保存在数据库中，模型输出和中间结果可存放于 `checkpoints/` 等目录。

在开展正式实验或论文复现实验前，建议：

- 明确数据集版本与数据预处理方式
- 固定训练配置与预测窗口参数
- 保存关键实验输出和评价指标
- 对不同实验任务使用清晰的命名规范
- 定期备份数据库、数据文件和模型检查点

## 注意事项

- 当前配置以开发环境为主，`DEBUG`、`SECRET_KEY`、`ALLOWED_HOSTS`、CORS 和数据库配置在生产环境中需要重新设置。
- SQLite 适合本地开发和轻量级实验，若用于多人协作或长期部署，建议迁移至 PostgreSQL、MySQL 等数据库。
- 大规模训练任务可能依赖 GPU、模型检查点和额外 Python 包，请根据实验环境单独配置。
- 项目中包含的示例数据、模型文件和实验结果应结合具体论文或实验说明进行管理。
